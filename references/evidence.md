# 实验与文献依据

## 一、四版本对照盲测（2026-07-12，本skill方法论的直接来源）

**设计**：取一篇AI辅助写成、读者反馈AI味重的公众号长文（约8500字，论文解读体）做用例。三个互相隔离的agent按不同方法改写，与原稿共四版，匿名乱序后交给四个认知距离拉开的读者persona（工程师老粉、实用派开发者、普通路人、重度AI疑心病的资深编辑）盲测。评委不知道哪版是原稿，也不知道实验假设。

**四个版本**：
- A 原稿：已有流程产物（词汇层禁用词已被hook拦干净）
- B 纯减法：结构手术（拆金句、删路标、破闭环、拉句长方差、hedge内嵌化），不加任何新素材
- C 减法+加法：结构手术+经验注入（从作者4577条真实社交动态里挖可溯源素材织入）
- D 流程反转：拆回观点卡片，重排成口述提纲，续写式成文

**结果**（像真人程度，1-10分均值）：

| 版本 | 均分 | 编辑评委估计的AI浓度 |
|------|------|---------------------|
| C 经验注入 | **8.6**（四票全胜） | 25-35% |
| D 骨架重建 | 6.0（深度/大众分裂） | 55-60% |
| B 纯减法 | 5.9（评语「干净得像消过毒」） | 70-75% |
| A 原稿 | 5.5 | 85-90% |

**三个关键发现**：
1. 纯减法只比原稿高0.4分——「删AI特征」的天花板被实测确认
2. C的决定性证据是「对论证毫无用处的真实枝蔓」（阿德勒的电车站、为一个判断押过的1700个token）——编辑评委的原话：「AI编细节是为论点服务的，不会把细节浪费在用不上的电车站上」
3. D在工程师（7分）和路人（5分）之间劈开：去掉小标题的「字墙」对大众读者是灾难。结构服务和经验注入是两件独立的事，前者要保留，后者要补上

**附带发现**：两个改写agent独立发现原稿中一处「个人场景」在素材库查无出处，疑似此前AI为了「加人味」编造。这是「绝不编造」铁律进入本skill的直接原因。

## 二、学术文献锚点

| 发现 | 来源 |
|------|------|
| 熟练AI用户识别AI文TPR 92.7%；把词汇线索洗掉后（paraphrase/humanize攻击）专家多数投票仍100%命中，靠原创性与结构 | Russell, Karpinska & Iyyer, 2025. [arXiv:2501.15654](https://arxiv.org/abs/2501.15654) |
| LLM反复套用句法模板（POS序列），换词不换句型；重复独立于n-gram存在 | Shaib et al., EMNLP 2024. [Syntactic Templates](https://aclanthology.org/2024.emnlp-main.368/) |
| LLM系统性偏好名词化、分词从句等语法构造，风格指令改不掉；instruct版比base版离人类风格更远 | Reinhart et al., PNAS 2025. [Do LLMs write like humans?](https://www.pnas.org/doi/10.1073/pnas.2422455122) |
| 人类句长分布显著更分散（burstiness），LLM概率曲线异常平坦 | Muñoz-Ortiz et al., 2024. [arXiv:2308.09067](https://arxiv.org/abs/2308.09067) |
| RLHF显著压缩输出多样性；根源是偏好标注的典型性偏差（人类标注者偏爱「典型」文本），模型学会拥抱中庸 | Kirk et al., ICLR 2024 [arXiv:2310.06452](https://arxiv.org/abs/2310.06452)；Verbalized Sampling [arXiv:2510.01171](https://arxiv.org/abs/2510.01171) |
| 与instruct模型合写显著拉平不同作者间的多样性，base模型不会 | Padmakumar & He, ICLR 2024. [arXiv:2309.05196](https://arxiv.org/abs/2309.05196) |
| few-shot模仿对松散个人化文体（博客、社交动态）基本无效——「参考我的旧文」不管用有学术解释 | Findings of EMNLP 2025. [arXiv:2509.14543](https://arxiv.org/abs/2509.14543) |
| 词汇级paraphrase把机器检测器击穿到4.6%检出率，但骗不过人类专家——词汇信号浅、结构信号深 | Krishna et al., NeurIPS 2023. [arXiv:2303.13408](https://arxiv.org/abs/2303.13408) |
| 「slop」的操作化定义：低信息密度+表面连贯，而非「有AI词」 | Shaib et al., 2025. [arXiv:2509.19163](https://arxiv.org/abs/2509.19163) |

## 三、实践圈的独立收敛

- Wikipedia「Signs of AI writing」（WikiProject AI Cleanup）：特征目录的母本，纪律是「看特征簇不看孤例；文风只是症状，病根是内容空洞」
- Gwern的system prompts：点名封杀「It was not X, but Y」与「No X, no Y—just Z」，要求「描述存在的东西」
- 中文社区的殊途同归：有作者攒了62条删AI腔规则、更新76次后承认「人味根本不是删出来的」；「人味儿写作心法」的方向是AI「不覆盖人」而非模仿人
- 三条路线口碑：AI初稿人重写（同质化重灾区）< 人写AI校对（安全但省时有限）< 人出骨架/口述+AI扩写（2026年共识方向）
